ğŸ” Algoritma Searching (Pencarian)

Searching adalah proses mencari keberadaan suatu nilai (key) dalam kumpulan data. Algoritma searching sangat penting untuk menemukan data dengan efisien dalam aplikasi nyata seperti database, search engine, dan sistem informasi.

ğŸ¯ 1. Linear Search (Sequential Search)

Konsep: Linear Search mencari data dengan memeriksa setiap elemen satu per satu dari awal hingga akhir array sampai data ditemukan atau array habis.

Analogi: Seperti mencari kunci di dalam tas dengan mengecek satu per satu barang di dalamnya sampai ketemu.

ğŸ“‹ Cara Kerja Linear Search

Langkah-langkah:

  1. Mulai dari elemen pertama (index 0)
  2. Bandingkan elemen saat ini dengan nilai yang dicari
  3. Jika cocok, kembalikan index elemen tersebut
  4. Jika tidak cocok, lanjut ke elemen berikutnya
  5. Ulangi hingga elemen ditemukan atau array habis
  6. Jika sampai akhir tidak ketemu, kembalikan -1 (tidak ditemukan)

ğŸ’» Implementasi Linear Search dalam C

#include <stdio.h>

int linearSearch(int arr[], int n, int key) {
    // Loop untuk memeriksa setiap elemen
    for (int i = 0; i < n; i++) {
        // Jika elemen ditemukan, kembalikan indexnya
        if (arr[i] == key) {
            return i;
        }
    }
    // Jika tidak ditemukan, kembalikan -1
    return -1;
}

void printArray(int arr[], int n) {
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

int main() {
    int arr[] = {5, 2, 8, 1, 9, 3, 7};
    int n = sizeof(arr) / sizeof(arr[0]);
    int key;
    
    printf("Data array: ");
    printArray(arr, n);
    
    printf("Masukkan angka yang dicari: ");
    scanf("%d", &key);
    
    int result = linearSearch(arr, n, key);
    
    if (result != -1) {
        printf("Data %d ditemukan di index %d\n", key, result);
    } else {
        printf("Data %d tidak ditemukan dalam array\n", key);
    }
    
    return 0;
}

Kompleksitas Linear Search:

  • Best Case: O(1) - data ada di posisi pertama
  • Average Case: O(n) - data di tengah-tengah
  • Worst Case: O(n) - data di posisi terakhir atau tidak ada
  • Space Complexity: O(1) - tidak butuh memori tambahan

âœ… Kelebihan Linear Search

  • Sederhana dan mudah diimplementasikan
  • Tidak memerlukan data yang terurut
  • Cocok untuk array kecil
  • Efisien jika data dicari ada di awal array

âŒ Kekurangan Linear Search

  • Lambat untuk array besar
  • Harus memeriksa semua elemen jika data tidak ditemukan
  • Tidak efisien dibanding Binary Search untuk data terurut

âš¡ 2. Binary Search

Konsep: Binary Search mencari data dengan membagi array menjadi dua bagian secara berulang, membandingkan nilai tengah dengan nilai yang dicari. SYARAT: Array harus sudah terurut!

Analogi: Seperti mencari kata dalam kamus - buka halaman tengah, jika kata yang dicari lebih kecil, cari di setengah kiri, jika lebih besar, cari di setengah kanan.

ğŸ“‹ Cara Kerja Binary Search

Langkah-langkah:

  1. Tentukan posisi awal (left) dan akhir (right) array
  2. Hitung posisi tengah: mid = (left + right) / 2
  3. Bandingkan elemen tengah dengan nilai yang dicari:
    • Jika sama, data ditemukan! Return mid
    • Jika nilai dicari < elemen tengah, cari di setengah kiri (right = mid - 1)
    • Jika nilai dicari > elemen tengah, cari di setengah kanan (left = mid + 1)
  4. Ulangi hingga data ditemukan atau left > right

ğŸ’» Implementasi Binary Search dalam C (Iteratif)

#include <stdio.h>

int binarySearch(int arr[], int n, int key) {
    int left = 0;
    int right = n - 1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2; // Hindari overflow
        
        // Jika elemen tengah adalah yang dicari
        if (arr[mid] == key) {
            return mid;
        }
        
        // Jika key lebih kecil, cari di setengah kiri
        if (arr[mid] > key) {
            right = mid - 1;
        }
        // Jika key lebih besar, cari di setengah kanan
        else {
            left = mid + 1;
        }
    }
    
    // Tidak ditemukan
    return -1;
}

void printArray(int arr[], int n) {
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

int main() {
    int arr[] = {1, 2, 3, 5, 7, 8, 9};
    int n = sizeof(arr) / sizeof(arr[0]);
    int key;
    
    printf("Data array (terurut): ");
    printArray(arr, n);
    
    printf("Masukkan angka yang dicari: ");
    scanf("%d", &key);
    
    int result = binarySearch(arr, n, key);
    
    if (result != -1) {
        printf("Data %d ditemukan di index %d\n", key, result);
    } else {
        printf("Data %d tidak ditemukan dalam array\n", key);
    }
    
    return 0;
}

ğŸ”„ Implementasi Binary Search (Rekursif)

#include <stdio.h>

int binarySearchRecursive(int arr[], int left, int right, int key) {
    // Base case: element tidak ditemukan
    if (left > right) {
        return -1;
    }
    
    int mid = left + (right - left) / 2;
    
    // Jika elemen tengah adalah yang dicari
    if (arr[mid] == key) {
        return mid;
    }
    
    // Jika key lebih kecil, cari di setengah kiri
    if (arr[mid] > key) {
        return binarySearchRecursive(arr, left, mid - 1, key);
    }
    
    // Jika key lebih besar, cari di setengah kanan
    return binarySearchRecursive(arr, mid + 1, right, key);
}

void printArray(int arr[], int n) {
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

int main() {
    int arr[] = {1, 2, 3, 5, 7, 8, 9};
    int n = sizeof(arr) / sizeof(arr[0]);
    int key;
    
    printf("Data array (terurut): ");
    printArray(arr, n);
    
    printf("Masukkan angka yang dicari: ");
    scanf("%d", &key);
    
    int result = binarySearchRecursive(arr, 0, n - 1, key);
    
    if (result != -1) {
        printf("Data %d ditemukan di index %d\n", key, result);
    } else {
        printf("Data %d tidak ditemukan dalam array\n", key);
    }
    
    return 0;
}

Kompleksitas Binary Search:

  • Best Case: O(1) - data ada di posisi tengah
  • Average Case: O(log n)
  • Worst Case: O(log n) - data di ujung atau tidak ada
  • Space Complexity: O(1) untuk iteratif, O(log n) untuk rekursif

âœ… Kelebihan Binary Search

  • Sangat cepat untuk array besar (O(log n))
  • Efisien - mengurangi area pencarian setengahnya setiap iterasi
  • Cocok untuk database dan sistem besar

âŒ Kekurangan Binary Search

  • Harus menggunakan array yang TERURUT
  • Jika data tidak terurut, harus di-sorting terlebih dahulu (overhead tambahan)
  • Lebih kompleks untuk diimplementasikan dibanding Linear Search

ğŸ† Kapan Menggunakan Algoritma Apa?

Gunakan Linear Search jika:

  • Array berukuran kecil (< 100 elemen)
  • Data tidak terurut dan tidak ingin/tidak bisa di-sorting
  • Mencari data yang kemungkinan ada di awal array
  • Implementasi sederhana lebih diprioritaskan

Gunakan Binary Search jika:

  • Array berukuran besar (> 100 elemen)
  • Data sudah terurut atau bisa di-sorting terlebih dahulu
  • Performa dan kecepatan adalah prioritas
  • Akan melakukan pencarian berulang kali pada data yang sama
